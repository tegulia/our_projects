# 蔬食料理课程学员管理与统计系统 - 使用说明

## 系统概述

蔬食料理课程学员管理与统计系统是一个专为"一期十次课"教学模式设计的Web应用程序。系统支持全课学员和单次学员的区分管理，自动记录出勤情况并智能计算剩余课时，为蔬食料理课程提供全面的学员管理解决方案。

## 界面介绍

### 1. 主页面布局

系统主页面分为以下几个部分：

- **顶部标题栏**：显示系统名称"蔬食料理课程学员管理与统计表"
- **统计看板**：展示四个关键统计指标
  - 全课学员总数
  - 单次学员总数
  - 本月签到次数
  - 平均出勤率
- **导航标签**：包含四个功能模块
  - 学员管理
  - 签到记录
  - 课程安排
  - 统计分析

### 2. 统计看板

统计看板位于页面顶部，实时显示系统的关键数据：

- **全课学员总数**：显示当前报名全课的学员数量
- **单次学员总数**：显示当前报名单次的学员数量
- **本月签到次数**：显示当月所有学员的签到总次数
- **平均出勤率**：显示整体课程的平均出勤百分比

## 使用方法

### 1. 学员管理

#### 添加学员

1. 点击"学员管理"标签页
2. 点击右上角的"添加学员"按钮
3. 在弹出的表单中填写以下信息：
   - **学员序号**：输入学员的唯一标识（如S001）
   - **姓名**：输入学员姓名
   - **联系电话**：输入学员的联系电话
   - **报名类型**：选择"全课"或"单次"
     - 全课：初始课时自动设为10次
     - 单次：初始课时自动设为1次
   - **购买课时**：根据报名类型自动填充，可手动调整
   - **备注**：可选，填写学员的额外信息
4. 点击"保存"按钮完成添加

#### 编辑学员

1. 在学员表格中找到要编辑的学员
2. 点击该学员所在行的"编辑"按钮（铅笔图标）
3. 在弹出的表单中修改学员信息
4. 点击"保存"按钮完成编辑

#### 删除学员

1. 在学员表格中找到要删除的学员
2. 点击该学员所在行的"删除"按钮（垃圾桶图标）
3. 在确认对话框中点击"确定"完成删除

### 2. 签到记录

#### 添加签到记录

1. 点击"签到记录"标签页
2. 点击右上角的"添加签到记录"按钮
3. 在弹出的表单中填写以下信息：
   - **学员ID**：从下拉列表中选择学员
     - 选择后，学员姓名会自动填充
   - **姓名**：自动填充，不可编辑
   - **签到日期**：默认为当前日期，可修改
   - **课程序号**：输入本次课程的序号（1-10）
   - **扣课类型**：选择"常规"或"补课"
     - 常规：扣减学员1次课时
     - 补课：不扣减课时
4. 点击"保存"按钮完成签到

#### 编辑签到记录

1. 在签到记录表格中找到要编辑的记录
2. 点击该记录所在行的"编辑"按钮（铅笔图标）
3. 在弹出的表单中修改签到信息
4. 点击"保存"按钮完成编辑

#### 删除签到记录

1. 在签到记录表格中找到要删除的记录
2. 点击该记录所在行的"删除"按钮（垃圾桶图标）
3. 在确认对话框中点击"确定"完成删除

### 3. 课程安排

#### 添加课程

1. 点击"课程安排"标签页
2. 点击右上角的"添加课程"按钮
3. 在弹出的表单中填写以下信息：
   - **课程序号**：输入课程的序号（1-10）
   - **课程日期**：选择课程日期
   - **课程主题**：输入本次课程的主题
   - **授课教师**：输入授课教师的姓名
4. 点击"保存"按钮完成添加

#### 编辑课程

1. 在课程表格中找到要编辑的课程
2. 点击该课程所在行的"编辑"按钮（铅笔图标）
3. 在弹出的表单中修改课程信息
4. 点击"保存"按钮完成编辑

#### 删除课程

1. 在课程表格中找到要删除的课程
2. 点击该课程所在行的"删除"按钮（垃圾桶图标）
3. 在确认对话框中点击"确定"完成删除

### 4. 统计分析

统计分析标签页提供两个可视化图表：

1. **学员类型分布饼图**
   - 显示全课学员和单次学员的比例分布
   - 图表会根据学员数据实时更新

2. **课程出勤情况柱状图**
   - 显示每门课程的出勤人数
   - 图表会根据签到记录实时更新

## 注意事项

### 1. 数据管理

- 所有数据保存在浏览器的本地存储中
- 刷新页面后数据不会丢失
- 清除浏览器缓存会导致数据丢失，建议定期备份重要数据

### 2. 课时管理

- 全课学员初始课时为10次
- 单次学员初始课时为1次
- 常规签到会扣减学员1次课时
- 补课签到不会扣减课时
- 学员课时不足时无法进行常规签到

### 3. 学员ID管理

- 学员ID是学员的唯一标识，不可重复
- 建议使用统一格式（如S001、S002等）
- 学员ID一旦创建，不建议修改

### 4. 课程序号

- 课程序号范围应为1-10，对应"一期十次课"的教学模式
- 建议按顺序设置课程序号，便于统计和管理

### 5. 浏览器兼容性

- 推荐使用Chrome、Firefox或Edge等现代浏览器
- 不支持IE浏览器
- 确保浏览器已启用JavaScript

### 6. 数据备份

- 虽然系统使用本地存储保存数据，但仍建议定期备份重要数据
- 可以通过浏览器的开发者工具导出本地存储数据

## 常见问题解答

### Q1: 为什么无法添加学员？

A1: 请检查所有必填字段是否已填写完整，特别是学员ID是否与现有学员重复。

### Q2: 为什么无法进行签到？

A2: 请检查学员剩余课时是否充足，课时不足的学员无法进行常规签到。

### Q3: 数据会保存在哪里？

A3: 所有数据保存在浏览器的本地存储中，不会上传到任何服务器。

### Q4: 如何查看历史数据？

A4: 系统会保存所有历史记录，可以在相应的标签页中查看完整的历史数据。

### Q5: 如何重置系统数据？

A5: 可以通过清除浏览器本地存储来重置系统数据，但请注意这将删除所有数据。
